{% raw %} About

Weibo Sensor is an application developed by the collaboration of UTRC China and TONGJI University. This application aims to automatically retrieve data from Sina Weibo, store and process these data, in order to finally display them with advanced visualization options. The collected are the following: statuses (Weibo microposts), comments, account profiles.

Functionalities This section presents an overview of the functionalities offered by the application
1. Home

When you connect to the application, you are redirected to the home page. This page displays some of the recent statistics, about the retrieved data, thanks to three different modules.

Last statuses retrieved: display the three last statuses (from the past 30 days) which contains at least one of the preset keywords combinations. By default will display every statuses which contains电梯or 扶梯. For each status the following information are displayed:

  • Account name
  • Publication date
  • Status’ message
  • Link to the account profile information
  • Link to get more information about the status

Amount of statuses retrieved: show the total amount of statuses retrieved in the past 30 days (comments are not included). This amount is compared to a global goal on a dashboard which allow to highlight the recent activity on the social media.

Topics relative activity: compare the relative activity (amount of statuses published) between the different preset topics on a doughnut chart. By default the topics are generated as followed:

  • 事故: contains事故 and (电梯 or扶梯)
  • 安全: contains安全and (电梯 or扶梯)
  • 维保: contains维保and (电梯 or扶梯)
  • 问题: contains问题and (电梯 or扶梯)
  • 服务: contains服务and (电梯 or扶梯)



2. Login/Logout

In order to be able to access the other menu, you should first log into the application, thus you should first click on the login icon (in the navigation bar). A popup will appear and you will be able to input your user name and password. Then, if both are correct, you will be able to access all the content of the application.

When you have finish to use the application you can logout by clicking on the logout icon (in the navigation bar).



3. Search

Searching data is one of the main functionality of the application. You can first chose the data you want to search by selecting the corresponding submenu, then you can input several search parameters and finally chose how you want to display the results.

3.1 Quick Search

This searching method allow to input one unique search parameter: keywords (to input several keyword they should be separated by a space “ ”). It will look for every statuses or comments which contains all of the keywords in the status/comment text and account name. The search results will be limited to the data published in past 30 days. Once the search button is pressed, result will be display in a visualization list.

Go to Quick Search

3.2 Search Status & Comment

As indicated, this second search method is used to search statuses and comments. Contrary to the quick search method, several search parameters can be input:

  • Keywords: allow to input keywords. Just as for quick search, several keyword can be input, they should be separated by a space “ ”. Every status or comment obtained will contain all of the keywords in the status/comment text and account name.
    Input example: “电梯 安全”.
  • Location: allow to input one keyword relative to a location. Will search only statuses and comments which account location contain this keyword.
    Input example: “上海”.
  • Mentions: allow to input one account name. Will search only statuses and comments which contain a reference to this account @USER. Warning: do not write the “@”.
    Input example: “奥的斯”.
  • Hashtags: allow to input one hashtag. Will search only statuses and comments which contain this #hashtag#. Warning: do not write the “#”.
    Input example: “电梯”.
  • Time period: allow to select two date. Will search only statuses and comments which were published in this time period. When you click on the calendar icon a pop up which allow to pick a date is opened. Warning: the largest will be this time period the longer it will take to obtain the results.
    By default the time period is set on the last 30 days.
  • Status feeling: allow to select between three sentiments. Will search only statuses and comments which correspond to the selected sentiments. If every sentiment or no sentiment are selected this parameter will not be applied.
  • Source: allow to select between five categories: Weibo website, IPhone, Android, Windows Phone, Others. Will search only statuses and comments which correspond to the selected categories. If every category or no category are selected this parameter will not be applied. Warning: this parameter is not fully implemented yet, thus results will probably not be accurate.
  • Status only: allow to select to only search statuses. If selected no comments will be obtained.
    By default is not selected.
  • Topics: not implemented yet.
  • Events: not implemented yet.

Results can be displayed in a visualization list or as statistics.

Go to Search Status & Comment

3.3 Search user

This third method is used to search account profiles information. The following parameters can be applied:

  • Name: allow to input a keyword. Will search only accounts which name contain this keywords.
    Input example: “电梯”.
  • Location: allow to input one keyword relative to a location. Will search only accounts which location contain this keyword.
    Input example: “上海”.
  • Followers: allow to input a number and select superior at“<” or inferior at “>” (click on the icon to switch between the two). Will search only accounts which total followers amount is superior/inferior at this number.
    Input example: “< 5000”.
  • Category: allow to select between several categories. Will search only accounts which correspond to the selected categories. If every category or no category are selected this parameter will not be applied.
  • Gender: allow to select between male, female or both. Will search only accounts which correspond to the selected gender.
  • Rate: not implemented yet.

Results will be displayed in a visualization list.

Go to Search User

3.4 Search Event

The last searching method, is used to search events. Several search parameters can be applied:

  • Keywords: allow to input keywords. Several keyword can be input, they should be separated by a space “ ”. Every event obtained will be linked to all of these keywords.
    Input example: “电梯 安全”.
  • Location: allow to input one keyword relative to a location. Will search only events which location is linked to this keyword.
    Input example: “上海”.
  • Time period: allow to select two date. Will search only events which were published in this time period. When you click on the calendar icon a pop up which allow to pick a date is opened. Warning: the largest will be this time period the longer it will take to obtain the results.
    By default the time period is set on the last 30 days.

Results will be displayed in a visualization list.

Go to Search Event


4. Statistics

Statistics provide an efficient way to visualize several data in one single chart. This section introduce the four charts which have been implemented in the version 1.0 of the application. Future version should implement more way to visualize data.

4.1 Time Statistics

This first chart provide a way to visualize the distribution of data on a period of time. By default it displays every statuses and comments published during the last 30 days, as shown here. Abscissa show the distribution of every days contained in the period of time targeted, and ordinate show the amount of statuses and comments for these days.

In order to display other data, you should use the Search Status and Comments page. From there several parameters can be applied, and then, instead of pressing the Search button, you should press Statistics and select Time option.

For example you can generate a chart with only data which contain 电梯, are negative and have been published on a specific period of time.



4.2 Sentiment Statistics

Warning: In the version 1.0 comments’ sentiments are not calculated yet.

There is two chart which allow to visualize the distribution of sentiments on a period of time. By default they display every statuses and comments published during the last 30 days.

As shown here, the first version displays the relative sentiment: difference between the amount of positive and negative statuses and comments, ignore neutral data. Abscissa show the distribution of every days contained in the period of time targeted, and ordinate show the amount of positive data, amount of negative data, and difference between these two amounts.

The second version displays every amount of data for each sentiment (negative, neuter, and positive). Abscissa show the distribution of every days contained in the period of time targeted, and ordinate show the amount of positive, neutral, and negative data.

In order to display other data, you should use the Search Status and Comments page. From there several parameters can be applied, and then, instead of pressing the Search button, you should press Statistics and select Sentiment option.



4.3 Location Statistics

As shown here, this chart display the distribution of data among Chinese provinces. By default they display every statuses and comments published during the last 30 days.

In order to display other data, you should use the Search Status and Comments page. From there several parameters can be applied, and then, instead of pressing the Search button, you should press Statistics and select Location option.



Data visualization and analysis
5.1 List of data visualization

After searching data, results are shown in a visualization page. Owing to the amount of data which may be displayed, the total amount of items on each page is limited. In order to show more items you should use the pagination bar. Each items will show a preview of the associated data (Status, comments, user account profiles, or events), and contain link to the associated analysis page (see bellow).

Statuses and comments visualization pages, have an additional feature which allow to save the data which are displayed: You should first click on Save results, then you should be able to select some of the item shown on the page (to select all in one click press on Select all on the top). Then you can chose to save this results as event or to directly download them as JSON (if download to JSON you can also select the field you want to collect).



5.2 Status analysis

This page is used to analyze both statuses and comments. Data are displayed as shown in the follwing picture (version is slightly different for comments):

  1. Type of data, Status or Comment
    • Name of whom published the message and link to his profile.
    • Publication date
    • Text of the status/comment
    • Source of the status/comment (which device were used)
  2. General information
    • Weibo: link to the corresponding Weibo page (disabled)
    • Reposts: show the list of the reposted message (if they have been retrieved)
    • Comments: show the list of the comments
    • In reply of: show the original status/comments (disabled if not)
    • Like: show the amount of like (does not link to any page)
  3. Link to other information
    • Topic
    • Related events
    • Keywords
  4. Calculated data
    • Location of the user profile when the message was posted
    • Map will show this location in future version
  5. Location
    • Positive: score between 65 and 100
    • Neutral: score between 35 and 65
    • Negative: score between 0 and 35
  6. Sentiment
  7. Estimate Geolocation, open a popup when pressed, will call some API to try to extract some location name from the text. This Feature is implemented as demonstration.


5.3 User analysis

This page is used to analyze user account profiles information. Data are displayed as shown in the follwing picture.

  1. Main information
  2. Links
    • Weibo link to this user profile page on Weibo
    • Website link to this user website (disabled if no link available)
  3. Location
  4. Other information
  5. Link to every statuses an comments published by this user
  6. Account category of this user (may be inaccurate on this version)
  7. User’s score (not implemented on this version)


5.4 Event analysis

This page is used to analyze events information. It will display the following information:

  • Name of this event
  • Description
  • Starting & stopping date
  • Location of this event
  • Keywords associated to this event
  • A link to visualize every related statuses and comments


Advanced functionalities
1 Manage

This version only implement the data cleaning management panel. This panel allow to visualize the cleaning tasks which are currently running and to add new tasks. In order to add a new tasks you should press the add button. Then, you will chose a name for this tasks, data will always come from the raw_data_db but you can select the destination database were your cleaning data will be saved. Yet the application only allow to clean by keywords, thus you should input a keyword combination. Finally you can chose when the task will be run, the current version allow only to use cyclic scheduling which can be triggered after a fix time period or at a specific time every day. You finally can create and start cleaning by pressing the final button.



2 Settings

No settings are available. Future versions will allow to change some of the user interface settings such as:

  • Change the keywords used to generated the home page chart
  • Modify the limitation on item per visualization page
